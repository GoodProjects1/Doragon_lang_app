# 🐉 Doragon · Deutsch

Un corso interattivo per **imparare il tedesco da zero**, pensato per italiani.
Tutto in un unico file HTML: niente installazioni, niente server.

Ogni giorno contiene 10 sezioni: **Lesen** (lettura), **Artikel** (articoli leggibili), **Hören** (dialoghi), **Musik** (canzoni con player YouTube), **Sprechen** (parlato col microfono), **Grammatik** (regole + esercizi), **Wortschatz** (vocaboli in frasi), **Kreuzworträtsel** (cruciverba), **Redewendungen** (modi di dire) e **Notizen** (note e domande da scrivere).
I livelli vanno da **A1 a B2**.

---

## 🚀 Come pubblicarlo su Internet (GitHub Pages, gratis)

Non serve essere programmatori: bastano pochi clic dal browser.

### 1. Crea un account GitHub
Vai su [github.com](https://github.com) e registrati (è gratuito).

### 2. Crea un nuovo repository
- In alto a destra clicca su **+** → **New repository**.
- **Repository name:** scrivi un nome, per esempio `doragon-deutsch`.
- Scegli **Public**.
- Clicca **Create repository**.

### 3. Carica i file
- Nella pagina del repository clicca **Add file** → **Upload files**.
- Trascina dentro **`index.html`** (e, se vuoi, anche questo `README.md`).
- In basso clicca **Commit changes**.

### 4. Attiva GitHub Pages
- Nel repository vai su **Settings** (in alto).
- Nel menu a sinistra clicca **Pages**.
- Sotto **Source** scegli **Deploy from a branch**.
- In **Branch** seleziona **main** e la cartella **/ (root)**, poi **Save**.

### 5. Apri il tuo sito 🎉
Dopo circa 1 minuto, ricarica la pagina **Settings → Pages**: comparirà il link del tuo sito, del tipo:

```
https://TUO-NOME.github.io/doragon-deutsch/
```

Aprilo: il corso è online e funziona su computer e telefono.

## 🔄 Come aggiornarlo in futuro
Quando avrai una versione nuova di `index.html`:
- Vai nel repository → clicca su **index.html** → icona della **matita** (✏️) → **Upload** della nuova versione,
  oppure **Add file → Upload files** e sostituisci quello vecchio → **Commit changes**.
- Il sito si aggiorna da solo dopo poco.

## 💡 Note tecniche (importanti)

- **Microfono (sezione Sprechen) e audio** funzionano al meglio su **Google Chrome** o **Microsoft Edge**.
  Su GitHub Pages il sito è in **HTTPS**, quindi il microfono è autorizzato senza problemi
  (la prima volta il browser chiede il permesso: clicca **Consenti**).
- Il **riconoscimento vocale** è ottimo su Chrome/Edge, più limitato su Safari/Firefox.
- Le **note** che scrivi nella sezione *Notizen* e i tuoi progressi (XP, livello, streak)
  vengono **salvati nel tuo browser**, su quel dispositivo.
- Il **player YouTube** carica il video solo quando premi ▶ (per non consumare dati inutilmente).
